## 🚀 IAP 패키지 기반 개발 우선순위 재조정 프레임워크

개발 우선순위는 **"사용자에게 제공하는 가치(Value)"**와 **"달성 가능한 수익 잠재력(Monetization Potential)"**에 따라 결정되어야 합니다.

### 1. IAP 티어별 핵심 목표 정의 (가설 설정)

각 티어가 사용자에게 어떤 경험을 제공하여 지불할 가치가 있는지 명확히 정의합니다.

| 티어 | 핵심 가치 제안 (Value Proposition) | 기대하는 사용자 행동 (KPI Focus) |
| :--- | :--- | :--- |
| **Basic** | 무료 사용자 유입 및 초기 체험 확보 (Engagement 극대화) | 일일 접속률 (DAU), 세션 수 |
| **Pro** | 편의성/몰입감 향상 (핵심 게임 루프 가속화, 미적 요소 강화) | 리텐션(Retention), 세션당 평균 플레이 시간 |
| **VIP** | 독점 경험 및 프리미엄 지위 (희소성, 독점 아트 에셋, 광고 제거 등) | LTV(고객 생애 가치), 전환율(Conversion Rate) |

### 2. 티어별 필수 기능 정의 (Feature Mapping)

위의 가치 제안을 달성하기 위해 UI/UX에 반드시 포함되어야 할 핵심 기능을 정의합니다. 이는 Coda-ri와 Designer가 구현해야 할 **최소 요구 사항**이 됩니다.

| 티어 | 필수 포함 기능 목록 (MVP 기준) | 비주얼/네온 적용 범위 |
| :--- | :--- | :--- |
| **Basic** | 기본 게임 플레이, 제한된 보상 시스템 활성화 | Neon_Vibe_Palette의 기본 UI 요소만 적용 |
| **Pro** | 고급 설정 옵션 (예: 조준선 커스터마이징), 무제한 리소스/시간 제공, 광고 제거 기능 | 네온 효과를 심화하고 동적 애니메이션 추가 |
| **VIP** | 독점 테마/아트 에셋 잠금 해제, 특별한 인게임 보상 시스템(독점 아이템), 광고 완전 제거 및 프리미엄 UI 레이어 적용 | 가장 복잡하고 화려한 네온 미학의 최고 레벨 구현 |

### 3. 개발 우선순위 조정 (ROI 기반)

개발 리소스가 한정되어 있으므로, **가장 낮은 노력으로 높은 수익을 창출할 수 있는 경로**를 최우선으로 설정합니다.

1.  **1순위: Basic $\rightarrow$ Pro 전환점의 명확화 및 구현 (Engagement $\rightarrow$ Retention)**
    *   **이유:** 무료 사용자를 유료 사용자(Pro)로 전환시키는 'Pain Point' 또는 'Gain Point'를 가장 빠르게 정의하고 구현해야 초기 매출 흐름을 확보할 수 있습니다.
2.  **2순위: Pro $\rightarrow$ VIP 전환점의 시각적 차별화 (Retention $\rightarrow$ LTV)**
    *   **이유:** Pro에서 VIP로 넘어갈 때 사용자가 느끼는 '프리미엄 보상'의 가치가 극대화되어야 하며, 이는 Designer가 정의한 네온 미학을 가장 화려하고 희소하게 구현하는 단계입니다.
3.  **3순위: 핵심 시스템 안정화 (Foundation)**
    *   **이유:** 모든 IAP 기능 구현 전에, 기존 코드의 오류 수정 및 API 통합(Coda-ri의 역할)을 완료하여 수익화 로직이 무너지지 않도록 기반을 다집니다.
