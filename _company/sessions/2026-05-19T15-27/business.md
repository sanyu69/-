# 📊 수익화 및 UX KPI 프레임워크 설계안 (Phase 2 목표)

## 1. 핵심 가설 설정 (Hypothesis Setting)

모든 측정 지표는 다음의 핵심 가설을 검증하는 데 초점을 맞춥니다.

*   **가설 A (수익화):** 시각적 차별화 요소(GLOW/Dynamic Effects)가 적용된 사용자 그룹은 비적용 그룹보다 IAP 구매 전환율이 통계적으로 유의미하게 높을 것이다.
*   **가설 B (사용자 경험):** VIP 티어 사용자는 Basic 티어 사용자보다 세션당 체류 시간이 길고, 게임 내 핵심 상호작용(예: 레벨업, 보스전)에 더 높은 참여율을 보일 것이다.

## 2. 핵심 성과 지표 (KPIs) 정의 및 측정 방안

측정할 KPI를 **수익화 지표**, **참여/사용성 지표**, **시스템 안정성 지표** 세 가지 카테고리로 분류하고, 각 지표에 대한 구체적인 측정 방법(Metric & Measurement Method)을 설계합니다.

| 카테고리 | 핵심 KPI (Metric) | 정의 (Definition) | 측정 방안 (Measurement Method) | 연관 전략 |
| :--- | :--- | :--- | :--- | :--- |
| **수익화 지표** | **Tier Conversion Rate (TCR)** | 특정 IAP 티어를 구매한 총 사용자 수 / 해당 티어에 도달한 세션 수 | 웹사이트 분석 툴(예: Google Analytics, 자체 로깅)에서 `purchase_complete` 이벤트 발생 시점의 사용자 그룹 분류 | 수익화 모델 검증 |
| | **ARPU (Average Revenue Per User)** | 특정 기간 동안 사용자 1인당 평균 매출액 | (총 매출액) / (총 활성 사용자 수) | 가격 전략 검증 |
| | **LTV (Lifetime Value)** | 한 사용자가 플랫폼에 기여하는 총 예상 수익 | 사용자별 누적 구매 금액 예측 모델 기반 산출 | 장기 수익화 검증 |
| **참여/사용성 지표** | **Session Duration (SDT)** | 평균 세션당 게임 플레이 시간 | 서버 로그에서 각 세션 시작 및 종료 시점의 타임스탬프 차이 계산 | UX 개선 방향 설정 |
| | **Feature Engagement Rate (FER)** | 특정 IAP 기능(예: 고급 효과 적용)을 활성화한 사용자 비율 | `feature_X_activated` 이벤트 발생 시, 해당 사용자의 세션 기록 분석 | 비주얼 전략 검증 |
| | **Retention Rate (D7/D30)** | 7일/30일 후 재방문 또는 재구매율 | 사용자 행동 로그를 기반으로 설정된 기간 내 활동 여부 측정 | 장기 고객 유지 검증 |
| **시스템 안정성 지표** | **Transaction Success Rate** | 성공적으로 완료된 결제 거래 수 / 시도된 결제 요청 수 | `purchase_success` 및 `payment_error` 로깅 데이터 분석 | $R_{Stability}$ 지속 확인 |
| | **Latency (API Response Time)** | IAP 관련 API 호출의 평균 응답 시간 | 백엔드 로그에서 API 엔드포인트별 응답 시간 측정 | 시스템 안정성 유지 |

## 3. KPI 대시보드 설계 (Dashboard Blueprint)

측정된 데이터를 시각화하여 의사결정을 지원하는 핵심 대시보드를 다음 세 가지 섹션으로 구성합니다.

1.  **수익 흐름 섹션:**
    *   월별/주별 ARPU 추이 (라인 차트)
    *   티어별 전환율 비교 막대 그래프 (Tier Conversion Rate)
    *   총 누적 매출 및 LTV 예측치 (단일 지표 카드)
2.  **사용자 행동 섹션:**
    *   평균 세션당 체류 시간 분포 (히스토그램 또는 박스 플롯)
    *   티어별 평균 게임 플레이 시간 비교 (SDT by Tier)
    *   주요 기능 활성화율 (FER) 대시보드
3.  **시스템 건전성 섹션:**
    *   최근 7일간의 결제 성공률 및 오류 발생 빈도 (Gauge 또는 알림 표시)
    *   IAP 관련 API 응답 지연 시간 평균 (Latency)

## 4. 데이터 수집 메커니즘 설계 (Data Collection Mechanism)

KPI를 정확히 측정하기 위해 웹사이트/백엔드 로직에 다음의 이벤트 로깅을 의무화합니다.

1.  **트랜잭션 로깅:** 모든 IAP 구매 시도, 성공, 실패, 환불 요청에 대해 사용자 ID, 티어 정보, 타임스탬프를 DB에 기록합니다. (TCR, ARPU 측정 기반)
2.  **행동 로깅:** 사용자가 게임 내에서 특정 상호작용(예: 보스 클리어, 특수 효과 활성화 버튼 클릭)을 완료할 때마다 해당 세션의 지속 시간과 상태 변화를 기록합니다. (SDT, FER 측정 기반)
3.  **시스템 로깅:** 모든 IAP API 호출에 대해 응답 시간을 기록하고, 시스템 오류 발생 시 상세 로그를 남깁니다. ($R_{Stability}$ 및 Latency 측정 기반)
