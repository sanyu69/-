## 🚀 최적 기술 스택 제안 (The Optimal Stack)

웹 기반 캐주얼 액션 게임에 가장 적합한 스택은 **'최소한의 오버헤드로 최대의 시각 효과를 내는 것'**입니다. 따라서 프레임워크 의존도를 최소화하고, 순수 브라우저 API 활용을 중심으로 제안합니다.

### 🥇 1순위 추천: Vanilla JavaScript + HTML Canvas (The Control Stack)

이 스택은 **성능, 통제력, 배포 용이성** 측면에서 가장 우수합니다.
*   **장점:**
    *   **최고의 성능 제어:** 게임 루프와 렌더링을 직접 제어하므로 불필요한 오버헤드가 없습니다. 이는 네온 효과나 복잡한 입자 시스템 구현에 절대적으로 유리합니다.
    *   **경량성:** 외부 라이브러리 의존성이 없어 빌드 과정이 간소화되며, 최종 파일 크기가 작습니다 (PWA 환경에도 적합).
    *   **직접적인 이해:** 모든 픽셀과 속도가 어디서 오는지 명확히 알 수 있어 디버깅이 매우 용이합니다.
*   **단점:** 구조를 잡기 위한 초기 설정(게임 루프, 충돌 감지 로직)을 직접 구현해야 합니다.

### 🥈 2순위 대안: Phaser.js (The Structured Stack)

프로토타입의 속도를 극대화하고 싶다면, 검증된 2D 게임 프레임워크인 Phaser를 고려할 수 있습니다.
*   **장점:**
    *   **개발 속도:** 물리 엔진, 스프라이트 관리, 씬(Scene) 관리가 내장되어 있어 초기 개발 속도가 매우 빠릅니다.
    *   **커뮤니티 지원:** 문제 발생 시 참고할 자료가 풍부합니다.
*   **단점:** Vanilla JS보다 상대적으로 오버헤드가 있으며, 네온 특유의 독특한 렌더링 효과를 커스터마이징할 때 추가적인 노력이 필요할 수 있습니다.

### 💡 최종 권고: **Vanilla JS + Canvas (초기 프로토타입)**

현재 목표가 '최적화된 시각 경험'과 '미니멀리즘'에 있다면, 복잡한 프레임워크 대신 **Canvas**를 사용하여 핵심 메커니즘(자동 이동, 파워업 선택, 충돌)을 직접 구현하는 것이 장기적으로 더 큰 통제력과 최적화를 제공할 것입니다.

## 🏗️ 초기 프로토타입 구조 제안 (Prototype Structure)

프로토타입은 **단일 HTML 파일**로 구성하여 즉시 실행 가능하고 배포 용이성을 확보해야 합니다. 이는 PWA(Progressive Web App) 환경에 가장 적합합니다.

### 📂 디렉토리 구조 (Project Root: `index.html` 중심)

```
/neon-replay-prototype
├── index.html          <-- 전체 게임 로직 및 UI 정의 (핵심)
├── style.css           <-- 네온 스타일, 레이아웃 정의
└── script.js           <-- 모든 게임 로직, 렌더링 루프 (Vanilla JS 구현)
```

### ⚙️ 핵심 파일별 역할 분담

1.  **`index.html` (The Container & UI)**
    *   게임 캔버스(`canvas` 태그)를 정의합니다.
    *   외부 리소스(CSS, JS 파일)를 로드합니다.
    *   게임 시작/종료 화면 및 네온 UI 요소를 담는 HTML 구조를 담당합니다.

2.  **`style.css` (The Vibe Setter)**
    *   전체 배경에 적용될 **어두운 배경색(Dark Mode)**과 **네온 색상 정의**를 설정합니다.
    *   캔버스 및 UI 요소에 대한 기본적인 시각적 스타일을 정의하여 아트 디렉션(`Designer`의 작업)을 구현합니다.

3.  **`script.js` (The Engine & Logic)**
    *   **Game Loop:** `requestAnimationFrame`을 사용하여 부드러운 게임 루프를 관리합니다.
    *   **Input Handler:** 마우스/키보드 입력을 받아 캐릭터의 방향이나 파워업 선택을 처리합니다.
    *   **Entity Management:** 적(Enemies), 플레이어, 아이템 등의 상태와 위치를 관리하는 클래스 또는 객체를 정의합니다.
    *   **Rendering Logic:** `Canvas API`를 사용하여 모든 엔티티를 캔버스에 네온 스타일로 그립니다.
    *   **Game State Management:** 점수, 생명력, 현재 활성화된 무기 등의 상태 변화를 추적하고 업데이트합니다.

### ✅ 초기 구현 시 우선순위 (Phase 1: Core Loop)

| 순서 | 목표 기능 | 상세 구현 내용 | 중요도 |
| :--- | :--- | :--- | :--- |
| **1** | **캔버스 및 루프 설정** | HTML에 Canvas 삽입, `requestAnimationFrame` 기반의 기본 게임 루프 시작. | ⭐⭐⭐ |
| **2** | **기본 엔티티 렌더링** | 플레이어와 적을 단순한 사각형이나 원으로 캔버스에 그리기 (네온 색상 적용). | ⭐⭐⭐ |
| **3** | **자동 움직임 구현** | 플레이어가 방향을 설정하면, 해당 방향으로 캐릭터가 지속적으로 이동하도록 로직 구현. | ⭐⭐ |
| **4** | **기본 충돌 감지** | 플레이어와 적 간의 간단한 AABB(Axis-Aligned Bounding Box) 충돌 처리. | ⭐⭐ |
| **5** | **파워업 시스템** | 일정 시간 생존 시, 무작위로 네온 효과를 부여하는 이벤트 트리거 구현. | ⭐ |
