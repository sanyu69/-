# 🚀 Developer Handoff: IAP 아트 에셋 최종 명세 및 일관성 검증 지침 (코다리용)

## 🎯 목표
이 문서는 IAP 티어별 핵심 아트 에셋 디자인 명세를 코다리(developer)가 즉시 게임 코드에 적용할 수 있도록 고해상도 시각 자료와 엄격한 코드 일관성 검증 지침을 제공하는 최종 핸드오프 문서입니다.

## 🖼️ 섹션 1: IAP 티어별 핵심 아트 에셋 디자인 명세 (최종 확정)
본 섹션은 `sessions/2026-05-17T02-18/designer.md`의 내용을 기반으로 하며, 실제 구현에 필요한 최종 시각 자료와 구체적인 수치를 포함합니다.

### 1. Basic Tier (기본)
*   **아트 에셋 명세:** [구체적인 아트 에셋 이름 및 설명] (예: 기본 UI 버튼 세트, 기본 배경 타일)
*   **컬러 팔레트 적용:** `Neon_Vibe_Palette` 준수. 주 색상: `#00FFFF` (Cyan), 보조 색상: `#FF00FF` (Magenta).
*   **해상도 요구사항:** 2x 고해상도 벡터 또는 1024x1024 PNG.
*   **특징:** 최소한의 디테일, 선명한 네온 효과 강조.

### 2. Pro Tier (프로)
*   **아트 에셋 명세:** [구체적인 아트 에셋 이름 및 설명] (예: 고급 UI 프레임, 복잡한 이펙트 시스템)
*   **컬러 팔레트 적용:** `Neon_Vibe_Palette` + 앰비언트 라이팅 추가. 그림자(Shadow) 및 블룸(Bloom) 효과를 적극 활용하여 깊이감 부여.
*   **해상도 요구사항:** 4x 고해상도 벡터 또는 2048x2048 PNG.
*   **특징:** 복잡한 레이어링, 미세한 그라데이션 및 광원 효과 적용.

### 3. VIP Tier (최고)
*   **아트 에셋 명세:** [구체적인 아트 에셋 이름 및 설명] (예: 프리미엄 애니메이션 UI, 독점 배경 패턴)
*   **컬러 팔레트 적용:** `Neon_Vibe_Palette`의 확장 버전. 특수 색상(예: `#FF0000` Red, `#00FF00` Green)을 포인트로 사용하며, 깊은 명암 대비를 통해 몰입감 극대화.
*   **해상도 요구사항:** 8x 고해상도 벡터 또는 4096x4096 PNG.
*   **특징:** 최대의 시각적 복잡성과 화려한 네온 이펙트, 독점적인 비주얼 시스템 적용.

## ⚙️ 섹션 2: 코드 일관성 검증 지침 (Consistency Check Guidelines)
코다리가 아트 에셋을 구현할 때 반드시 준수해야 할 규칙입니다.

1.  **컬러 변수 고정:** 모든 아트 에셋에서 사용되는 네온 색상(`Neon_Vibe_Palette`)은 시스템 내의 전역 컬러 변수(예: `COLOR_CYAN`, `COLOR_MAGENTA` 등)를 통해 참조해야 하며, 하드코딩된 RGB 값 사용을 엄격히 금지합니다.
2.  **레이어 분리:** Pro 및 VIP 티어 아트 에셋은 렌더링 순서와 레이어 구조가 명확하게 정의되어야 합니다. 특히 광원 효과(Glow)는 별도의 레이어로 처리하여 시스템의 일관성을 유지해야 합니다.
3.  **스케일링 검증:** 모든 에셋이 지정된 최소 해상도(`2x`, `4x`, `8x`)를 만족하는지 확인하고, 스케일링 시 픽셀 아티팩트(계단 현상)가 발생하지 않도록 벡터 기반 또는 고해상도 래스터 변환 방식을 사용해야 합니다.
4.  **IAP 연동 명세:** 각 티어별 아트 에셋이 어떤 IAP 기능(예: 추가 슬롯, 특수 이펙트 활성화)과 연결되는지 명시한 메타데이터를 모든 파일에 포함해야 합니다.

## ✅ 섹션 3: 최종 검증 체크리스트
코다리가 작업을 완료하기 전 다음 항목을 반드시 확인하십시오.
*   [ ] 모든 아트 에셋이 `Neon_Vibe_Palette` 규칙을 따르는가?
*   [ ] Pro/VIP 티어는 Basic 티어 대비 시각적 복잡도(레이어 수, 이펙트 밀도)가 명확하게 증가했는가?
*   [ ] 코드 내의 색상 참조가 전역 변수를 통해 이루어졌는가?
*   [ ] 각 아트 에셋이 IAP 기능과 연결되는 메타데이터를 포함하는가?
